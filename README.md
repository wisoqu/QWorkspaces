# QWorkspaces

QWorkspaces - небольшой прототип приложения на Flet с приветственным экраном и персональным рабочим пространством.

На текущем этапе приложение демонстрирует базовый пользовательский сценарий:
- пользователь вводит имя на стартовом экране;
- имя сохраняется в сессии;
- после перехода на главный экран открывается персональное рабочее пространство;
- внутри рабочего пространства можно добавлять временные заметки в рамках текущей сессии.

## Назначение проекта

Проект служит стартовой точкой для приложения с личным рабочим пространством, где пользователь проходит через простой экран входа и попадает в свою рабочую область.

## Текущие возможности

- стартовый экран с вводом имени;
- проверка на пустой ввод;
- навигация между экранами через роуты;
- хранение имени пользователя в сессии;
- экран рабочего пространства с боковой панелью;
- добавление заметок без сохранения в базу данных или файл.

## Стек

## Запуск

1. Создать и активировать виртуальное окружение.
2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Запустить приложение:

```bash
python src/main.py
```

## Что делают экраны

### Экран приветствия

Находится в `src/screens/hello.py`.

Отвечает за:
- запрос имени пользователя;
- проверку, что поле не пустое;
- сохранение имени в сессии;
- переход пользователя на `/home`.

### Главный экран

Находится в `src/screens/home.py`.

Отвечает за:
- отображение имени текущего пользователя;
- отображение боковой панели и основной области;
- добавление заметок в рамках текущей сессии.

## Текущие ограничения

- заметки не сохраняются после перезапуска;
- отсутствует система авторизации;
- пока нет автоматических тестов;
- в `requirements.txt` сейчас указана только базовая зависимость.

## Безопасность Git

Репозиторий настроен так, чтобы в коммиты не попадали локальные и чувствительные артефакты разработки:
- виртуальные окружения;
- локальные `.env`-файлы;
- папки IDE и служебных инструментов;
- кэши и артефакты сборки;
- локальные базы данных и типовые файлы сертификатов и ключей.

## Чек-лист перед следующим коммитом

Перед следующим коммитом стоит проверить:
- в staging попали только нужные изменения;
- локальные `.env`, `venv`, кэши и IDE-файлы не включены;
- сообщение коммита понятно описывает текущее состояние проекта.
